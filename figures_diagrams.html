Fig 6: Opening Criterion

Figure 6: Node opening criterion. (a) BVH uses maximum AABB extent s. (b) Traditional octree uses cell half-width h. Both compare s/d (or h/d) against threshold θ.

Fig 7: System Architecture

Figure 7: System architecture. Solid arrows show the GPU-primary data path; dashed arrows indicate periodic staging-buffer readback for CPU-side diagnostics. The rendering pipeline reads directly from GPU storage buffers.

Fig 8: Timestep Pipeline

Figure 8: Single-timestep pipeline (Leapfrog KDK). The LBVH build phase expands into six GPU compute passes separated by pipeline barriers. The dashed diagnostics readback occurs periodically, not every step.

Fig 9: LBVH Construction Pipeline

Figure 9: LBVH construction pipeline. Six GPU compute passes build the bounding-volume hierarchy each timestep. Dashed lines indicate pipeline barriers (storage-buffer read-after-write) between passes. Colors group related phases: scene analysis (blue), sorting (amber), topology (green), aggregation (purple).
